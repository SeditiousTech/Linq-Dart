Overview

Linq-Dart is a port of .NET's Linq Library tailored for manipulating and querying generic Dart collections. It's not a 1:1 port and it retains the .NET casing style instead of Dart's camelCase. You view the Documentation here or see below to see what's implemented:

Simple Examples

The library consists of one primary class: Queryable<T>. Queryable collections take an argument of a type that inherits from Collection<T> in the constructor:

var myCollection = getSomeKindOfGenericCollectionOfTypeT();
Queryable<T> myQueryable = new Queryable(myCollection);

Included in the source is a test class model called Pie. We can find all the pies that have a cost greater than $3 using:

Queryable<Pie> pies = new Queryable(Pie.GetTestPies());
pies.Where((p) => p.cost >= 3).ForEach((e) => print(e));

// Output:
//  Apple (3.29)
//  Cherry (4.29)
//  Blueberry (4.29)
//  Meat (5.7)

We can then filter that result to only get the two cheapest pies except the cheapest pie:

pies.Where((p) => p.cost >= 3)
    .Skip(1)
    .Take(2)
    .ForEach((e) => print(e));

// Output:
//  Cherry (4.29)
//  Blueberry (4.29)

The above example uses a pattern called method chaining — most of Queryable<T>'s methods returns a new instance of Queryable<T>.
